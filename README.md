# 项目资源管家

面向 Cocos Creator 3.8.6 的项目级编辑器扩展。当前提供资源扫描、资源引用检查、资源自动分类、批量移动、未引用资源清理、健康检查、移动历史、反向移动计划和报告导出。

## 使用方式

1. 在扩展管理器启用 `asset-steward`。
2. 点击顶部菜单 `项目资源管家 -> 打开资源管家`。
3. 在顶部 Tab 中选择功能页。
4. 当前可执行功能集中在“资源扫描”“自动分类”“健康检查”和“历史与报告”页。
5. 顶部“工具面板”可按项目保存功能开关；关闭后对应功能块会从页面隐藏。
6. 移动前先生成预览计划，人工复核后再执行。

## 资源管家 Tab

- `资源扫描`：独立展示缺失 `.meta`、孤立 `.meta`、空目录和类型统计；提供资源 UUID 静态引用检查；只预览和定位，不删除、不修复。
- `自动分类`：当前完整可用的资源扫描、规则分类、手动移动和计划执行。
- `未引用资源`：已接入主场景 UUID 依赖图的未引用候选扫描、筛选、定位、删除预览、强制备份和 AssetDB 删除执行；原独立扩展保持不变。
- `健康检查`：已接入场景节点引用检查、resources 动态加载检查、包体贡献统计、目录规范检查、材质贴图检查、重复资源检查和场景/Prefab 引用健康检查。
- `工具面板`：可开启或关闭健康检查中的功能块，例如隐藏 resources 动态加载检查；配置保存到项目 profile。
- `历史与报告`：当前支持移动历史详情、反向移动计划、运行日志和当前会话 Markdown/JSON 报告导出。

## 功能

- 扫描 `assets/` 下的文件和目录，不把 `.meta` 单独列为可移动项。
- 资源扫描页支持按目录、路径关键字、扩展名和异常忽略项筛选，并展示异常列表和类型统计；默认忽略 `.gitkeep` 这类占位文件的缺失 meta 异常。
- 资源扫描页展示可定位的资源列表，可直接点击“查引用”读取目标 `.meta` UUID，并反查 `.scene/.prefab/.mtl/.material/.anim/.effect` 等序列化资源引用方。
- 资源引用检查支持直接使用当前选中资源发起检查，并仍保留手动输入一个或多个资源路径的兜底方式。
- 引用方支持继续点击“查上级”，将当前引用文件作为新目标继续反查；`.scene/.prefab` 引用会逐条展示节点路径、组件和字段。
- 场景/Prefab 引用位置提供“定位资源”和“选中节点”；选中节点只在对应资源当前已打开且序列化节点 ID 可用时执行，不会自动切换场景。
- 健康检查页可根据当前选中节点或手动输入的节点 ID，扫描 `.scene/.prefab`，列出哪些组件属性引用了该节点，并显示组件所在节点路径、组件类型和字段。
- 健康检查页可对照 `assets/resources` 与 `assets/script`、`assets/scripts` 中直接调用的 `resources.load/loadDir`，列出疑似未加载资源、疑似缺失路径和动态调用告警。
- `resources.loadDir("目录")` 会将该目录下全部 resources 资源视为静态命中；变量、字符串拼接和封装调用只提示人工复核。
- 健康检查页可按扫描目录统计源资源磁盘体积，展示所有子目录递归大小、扩展名大小、Top N 大文件和 Top 主场景递归引用大文件；默认扫描 `assets`、排除 `.meta`、显示 Top 20。
- Top 主场景递归引用大文件从可配置主场景出发，沿序列化 UUID 依赖链递归统计，并展示一条引用链，例如 `主场景 -> Prefab -> 材质 -> 贴图` 或 `主场景 -> Prefab -> 模型`。
- 健康检查页可复用当前自动分类规则检查目录规范；默认扫描 `assets/res`，只列出首个命中规则下目录不符合的资源，并可一键生成现有移动预览计划。
- 健康检查页可扫描 `assets/res` 下全部 `.mtl/.material/.pmtl`，展示正常材质贴图关系、无法解析的待复核 UUID，并标记材质是否从主场景递归可达。
- 健康检查页可扫描不同目录下的同名资源，并对相同大小的候选文件分块计算 SHA-256，展示内容完全相同的重复资源组和理论重复体积。
- 健康检查页可扫描 `.scene/.prefab` 中的完整 UUID 引用，对照项目主资源和子资源 UUID，展示无法解析的待复核项、涉及文件和命中次数，并支持精确 UUID 白名单。
- 未引用资源页默认从 `assets/scene/main.scene` 扫描 `assets/res`，列出未从主场景序列化 UUID 依赖图到达的候选，并支持路径、扩展名筛选、定位和人工勾选。
- 未引用删除候选会在预览和执行前都按当前主场景依赖图重新校验；默认只备份勾选候选及 `.meta`，也可选择备份整个扫描目录。
- 未引用删除执行会先复制备份到 `backups/asset-steward/unused-delete-时间戳/` 并写入 `manifest.json`，备份成功后才通过 Creator AssetDB 删除。
- 搜索、扩展名筛选、全选当前结果、资源管理器定位。
- 手动把多个资源或目录移动到同一个已存在目标目录。
- 按扩展名和文件名关键词自动分类，规则按显示顺序首个匹配项生效；关键词为空表示该扩展名的兜底规则。
- 默认贴图分类为 `image/ui`、`image/vfx`、`image/environment`、`image/model` 和 `image/other`。
- 默认将 `.effect/.chunk` 分类到 `assets/res/effects`，将 `.mtl` 分类到 `assets/res/material`。
- 自动分类和反向计划可在执行时创建缺失目标目录并刷新 AssetDB。
- 冲突策略支持跳过、自动重命名和覆盖。
- 执行前预览源路径、目标路径、重命名、覆盖和阻止原因。
- 检测缺少 `.meta`、无效目标、移动到自身子目录、重复目标等风险。
- 孤立 `.meta` 只在资源扫描页显示路径，不作为普通资源定位或移动。
- 资源引用检查只做静态 UUID 搜索，未找到引用不代表资源一定可删除。
- 场景节点引用检查只解析已保存到磁盘的 `.scene/.prefab` 序列化 `__id__` 引用；未保存的场景改动、运行时动态绑定和脚本逻辑不会被识别。
- Prefab 实例覆盖和部分序列化对象可能没有稳定节点 ID，此时仍展示文件、字段和可解析位置，但只允许定位资源。
- 保存最近 30 次成功移动历史，并可生成反向移动计划。
- 可在移动成功后删除本次源路径产生的空目录及连续空父目录。
- 运行日志会记录扫描 warning、引用检查 warning、接口版本不匹配和执行失败，日志保存在 `profiles/asset-steward.logs.json`。
- 历史与报告页可将当前面板会话中已运行模块的最近结果、当前移动计划、移动历史摘要和运行日志一次性导出为 Markdown 与 JSON；文件保存到 `reports/asset-steward/`，并使用同一时间戳基名。
- 历史与报告页可查看最近 30 条移动历史中的单次完整成功移动列表、覆盖可恢复状态、删除空源目录数量和清理失败计数。
- 主进程提供 `get-toolbox-framework` 框架接口，面板使用统一模块清单渲染未引用资源、健康检查和报告入口，方便后续逐个接入功能。

## 安全限制

- 不允许移动 `assets` 根目录。
- 手动移动的目标目录必须已经存在；自动分类和反向计划可以创建缺失目录。
- 缺少 `.meta` 的资源会被阻止移动。
- 覆盖仅允许文件覆盖文件，执行前必须确认已经备份，并再次弹窗确认。
- 覆盖会先删除原目标资源，反向移动计划无法恢复被覆盖的原目标。
- 反向计划同样只是一份待复核计划，不承诺强制回滚。
- 执行期间若源文件或目标状态发生变化，该项会失败并要求重新扫描和预览。
- 空目录清理只检查本次成功移动资源的原目录，不扫描或删除整个项目中的其它空目录；缺少 `.meta` 的空目录会拒绝删除。
- `assets/res`、`assets/resources`、`assets/script`、`assets/scene` 等项目结构根目录不会被空目录清理删除。
- resources 动态加载检查和包体贡献统计只执行只读扫描，不移动或删除资源；源资源体积不等同于构建后包体。
- 主场景引用排行严格依据 UUID 依赖图；`resources.load/loadDir` 即使能静态识别也不计入。孤立材质不会让其下游贴图被计入，但共享贴图仍可能通过其它有效主场景链进入排行。
- 目录规范检查默认只报告；生成移动预览后仍受缺少 `.meta`、目标冲突、重命名、覆盖确认和 AssetDB 移动规则约束。
- 材质贴图检查默认只报告和定位；无法在项目 UUID 图中解析的贴图引用标记为“待复核”，不直接判定贴图丢失，也不自动修改材质。
- 重复资源检查默认只报告和定位；同名或相同 hash 不代表可以删除，处理前必须结合 UUID 引用、动态加载和玩法需求人工确认。
- 场景和 Prefab 引用健康检查默认只报告和定位；无法解析 UUID 不直接判定资源丢失，确认属于引擎内置资源后才能加入白名单。
- 会话报告导出只写入 `reports/asset-steward/` 文本文件，不重新运行扫描、不修改资源；当前移动计划的临时执行 token 不会写入报告。
- 历史详情只读取已持久化的移动历史；执行失败项和清理失败明细当前不补写旧记录，需要结合当次运行日志复核。
- 未引用资源候选不代表可删除；脚本与 Shader Chunk 强制保护，`resources.load/loadDir`、AssetManager、运行时拼接路径和渠道资源需要人工复核。删除执行必须先成功创建工具备份，并经过人工勾选和二次弹窗确认。
- 如果面板出现字段不完整提示，通常表示 Creator 中的扩展主进程未重载；请在扩展管理器重载 `asset-steward` 后重新打开面板。

建议移动前提交或备份项目，并在移动后使用 Cocos Creator 打开相关场景、Prefab 和材质做引用回归。
