# 项目资源管家 Feature 规划

本文档记录 `asset-steward` 后续作为“项目资源管家”的功能规划和历史设计背景。

日常使用流程见 [`docs/user-guide.md`](docs/user-guide.md)，开发维护说明见 [`docs/developer-guide.md`](docs/developer-guide.md)，发布变化见 [`docs/changelog.md`](docs/changelog.md)。

当前确认策略：

- 合并 `scene-unused-assets` 到资源管家，但先保留旧扩展不动。
- 允许删除能力，但必须预览、备份勾选和二次确认。
- 已实现资源扫描模块的低风险范围：资源列表、缺失 `.meta`、孤立 `.meta`、空目录检查，只预览和定位。
- 已实现资源引用检查第一版：扫描列表可一键选择资源，读取 `.meta` UUID，静态反查序列化资源引用方，只预览和定位。
- 已实现 resources 动态加载检查第一版：对照 `assets/resources` 和直接调用的 `resources.load/loadDir`，只报告和定位。
- 已实现包体贡献统计第一版：按所有子目录递归大小、扩展名大小、Top 文件和 Top 主场景递归引用文件展示源资源体积。
- 已实现目录规范检查第一版：复用自动分类规则，报告目录不符合项并支持转为移动预览。
- 已实现未引用资源能力：从主场景 UUID 依赖图生成候选，支持筛选、定位、删除预览、强制备份和 AssetDB 删除执行。

## 1. 资源管家框架

### 目标
- 用一个扩展承载项目资源扫描、分类、未引用资源、健康检查和报告能力。
- 保留现有自动分类和批量移动能力，不破坏已有规则和历史。

### 面板入口
- 菜单：`项目资源管家 -> 打开资源管家`
- 面板 Tab：
  - `资源扫描`
  - `自动分类`
  - `未引用资源`
  - `节点引用`
  - `resources 加载`
  - `包体统计`
  - `目录规范`
  - `材质贴图`
  - `重复资源`
  - `场景引用`
  - `历史与报告`

### 当前状态
- 已完成多 Tab 框架。
- 已迁移自动分类和历史反向计划。
- 已完成 `资源扫描` Tab 的低风险体检入口。
- 已完成资源引用检查第一版，并放在 `资源扫描` Tab 下方；扫描资源列表支持一键“查引用”。
- 已完成 resources 动态加载检查第一版，并拆为独立 `resources 加载` Tab；`loadDir` 命中的目录按整目录已使用处理。
- 已完成包体贡献统计第一版，并拆为独立 `包体统计` Tab；默认扫描 `assets`、排除 `.meta`、显示 Top 20。
- 已新增 `get-toolbox-framework` 框架接口，统一登记各 Tab 后续模块的状态、输入、输出、安全边界和下一步。
- 原健康检查中的 7 项能力已全部拆为独立功能页；危险操作仍走各模块的预览、确认、备份或审计流程。

## 2. 已落地能力索引

以下只保留稳定能力和安全边界的索引。具体操作流程以 [用户手册](docs/user-guide.md) 为准，模块拆分和维护约定以 [开发手册](docs/developer-guide.md) 与 [架构说明](docs/architecture.md) 为准。

- 资源扫描：检查缺失 .meta、孤立 .meta、空目录并展示资源列表，只预览和定位，不自动修复或删除。
- 自动分类：按规则生成移动预览，继续通过 AssetDB 移动资源，保留冲突策略、历史记录和反向计划。
- 未引用资源：从主场景 UUID 依赖图生成候选，支持筛选、定位、删除预览、强制备份和执行审计。
- 资源引用检查：读取目标资源 UUID，静态反查序列化资源引用方，并可逐级追踪上级引用。
- 独立检查功能：覆盖 resources 动态加载、包体贡献、目录规范、材质贴图、重复资源、场景/Prefab 引用健康和场景节点引用检查。
- 历史与报告：保留移动历史、失败明细、运行日志、Markdown/JSON 报告导出和反向计划入口。
- 安全边界：默认只读扫描；移动、覆盖、删除等危险操作必须经过预览、确认、备份或审计流程，不绕过 Creator AssetDB。

## 3. 实现顺序建议

### Feature 1：资源扫描 Tab
- 缺失 `.meta`。
- 孤立 `.meta`。
- 空目录。
- 大文件列表。
- 搜索和定位。
- 不执行删除。

### Feature 2：资源引用检查
- 选择资源反查引用方。
- 用 UUID 搜索序列化资源。
- 删除或覆盖前提供风险确认依据。
- 当前已完成第一版。

### Feature 3：resources 动态加载检查
- 扫描 `assets/resources`。
- 扫描代码中的静态加载路径。
- 标记可能未使用或可能加载失败的 resources 资源。
- 当前已完成第一版。

### Feature 4：包体贡献统计
- 目录大小排行。
- 类型大小排行。
- Top 大文件。
- 接入资源扫描报告。
- 当前已完成第一版。

### Feature 5：目录规范检查
- 复用自动分类规则。
- 生成建议移动目标。
- 可转为移动预览计划。
- 当前已完成第一版。

### Feature 6：合并未引用资源
- 迁移 `scene-unused-assets` 扫描逻辑。
- 保留旧扩展。
- 资源管家内提供扫描、定位、删除候选。
- 当前已完成扫描、筛选、定位、删除预览、强制备份和 AssetDB 删除执行。
- 删除候选默认不选中任何项；预览和执行前都会重新校验候选仍未从主场景 UUID 依赖图到达。
- 备份默认只复制勾选候选及 `.meta`，可选备份整个扫描目录；备份保存到 `backups/asset-steward/unused-delete-时间戳/` 并写 manifest。
- 备份成功、风险勾选和二次确认后才允许删除。

### Feature 7：材质贴图检查
- 默认扫描 `assets/res` 下全部 `.mtl/.material/.pmtl`。
- 展示正常材质贴图关系和无法解析的待复核引用。
- 标记材质是否从主场景递归可达。
- 只报告，不自动修复。
- 当前已完成第一版。

### Feature 8：重复资源检查
- 默认扫描 `assets/res` 下普通资源，不扫描 `.meta`。
- 展示不同目录下的同名资源组。
- 先按文件大小筛选候选，再分块计算 SHA-256，展示重复内容组和理论重复体积。
- 只报告，不自动删除。
- 当前已完成第一版。

### Feature 9：场景和 Prefab 引用健康检查
- 默认扫描 `assets` 下 `.scene/.prefab` 中的完整 UUID。
- 对照项目主资源和子资源 UUID，展示无法解析 UUID、涉及文件和命中次数。
- 支持逗号、分号或换行分隔的完整 UUID 精确白名单。
- 无法解析项只标记为“待复核”，不直接判定资源丢失。
- 资源整理后用于回归。
- 当前已完成第一版。

### Feature 10：报告和历史详情完善
- 当前面板会话中已运行模块的最近结果导出 Markdown。
- 与 Markdown 共用同一会话快照导出 JSON。
- 包含当前移动计划、移动历史摘要和运行日志。
- 保存到 `reports/asset-steward/`，Markdown/JSON 使用同一时间戳基名。
- 导出时不重新运行扫描，移动计划临时执行 token 不写入报告。
- 查看单次历史完整成功移动列表、覆盖风险和空目录清理结果。
- 反向计划执行结果记录。
- 当前已完成 Markdown/JSON 报告导出和历史详情第一版；反向计划执行结果记录暂未完成。

### Feature 11：场景节点引用检查
- 根据当前选中节点或手动输入的节点 ID，扫描 `.scene/.prefab`。
- 先在扫描范围内定位目标节点，再反查哪些组件属性通过序列化 `__id__` 引用该节点。
- 结果展示引用组件所在文件、组件节点路径、组件类型、字段路径和目标节点路径。
- 支持定位引用资源；当对应场景或 Prefab 已打开时，可选中引用组件所在节点或目标节点。
- 只报告已保存的序列化引用，不识别未保存改动、运行时动态赋值和脚本逻辑引用。
- 当前已完成第一版，入口为独立 `节点引用` Tab。

## 4. 不做或暂缓

- 不直接修改 `assets/scene/main.scene`。
- 不自动删除重复资源。
- 不自动修复缺失 `.meta`。
- 不扫描和整理 `library/`、`temp/`、`build/`。
- 不绕过 Creator AssetDB 直接删除 Cocos 资源。

## 5. 资源引用扩展

### 目标
- 在现有“目标资源 -> 直接引用方”结果上继续向上追踪，帮助确认资源最终由哪些场景、Prefab 或其它资源使用。
- 当引用发生在场景或 Prefab 内时，将结果精确到节点、组件和字段，而不是只展示引用方文件。

### 功能
- 引用方操作栏除“定位”外，新增“查上级引用”：
  - 对当前引用方再次执行引用检查，展示它的直接上级引用方。
  - 可继续查询上上级及更高层引用，行为与主场景递归引用链一致。
  - 每次查询替换当前目标，用户可按 `贴图 -> 材质 -> Prefab -> 场景` 顺序逐级追踪。
- 场景和 Prefab 引用详情：
  - 引用方为 `.scene` 或 `.prefab` 时，解析序列化数据中的引用位置。
  - 展示引用所在的节点名称和节点层级路径。
  - 展示引用所在的组件类型。
  - 展示组件中引用目标资源的字段路径。
  - 同一文件内存在多处引用时逐条展示，不只汇总命中次数。
- 引用详情示例：
  - `assets/scene/main.scene`
  - 节点：`Canvas/HUD/FishIcon`
  - 组件：`cc.Sprite`
  - 字段：`_spriteFrame`

### 输出
- 直接引用方列表。
- 每个引用方的上级引用入口，可重复操作逐级追踪引用链。
- 场景或 Prefab 内的节点路径、组件类型、字段路径和命中 UUID。
- 无法解析到具体节点或组件时，保留文件级结果并标记“仅文件级可定位”。

### 安全限制
- 只解析文本序列化资源中的静态 UUID 引用，不修改场景、Prefab 或资源。
- 代码动态加载、运行时拼接路径和引擎内部引用无法精确到节点、组件或字段，需要人工复核。
- “选中节点”只查询当前打开资源，不自动打开或切换场景/Prefab。
- Prefab 实例覆盖和部分序列化对象没有稳定节点 ID 时，只提供引用位置和资源定位。

### 当前状态
- 已按 `1A、2C` 完成第一版。
- 引用方支持逐级“查上级”；场景/Prefab 引用逐条展示节点、组件和字段。
- 已提供“定位资源”和当前打开资源内“选中节点”，不会自动切换场景。

## 6. 场景节点引用检查

### 目标
- 从“场景里的某个节点”反查有哪些组件属性引用了它。
- 帮助删除、重命名、移动或拆分节点前确认依赖位置。

### 功能
- 输入节点 ID；输入为空时使用当前 Creator 选中的节点。
- 扫描目录默认 `assets`，文件类型默认 `.scene,.prefab`。
- 在扫描到的场景或 Prefab 中按节点 `_id` 定位目标节点。
- 只统计组件对象中的字段引用，过滤组件自身 `_node` 挂载关系。
- 展示引用组件的节点路径、组件类型、字段路径和目标节点路径。
- 支持定位资源；对应资源已打开时，可选中引用组件所在节点或目标节点。

### 安全限制
- 只读解析磁盘上的 `.scene/.prefab` 序列化 JSON，不修改场景或 Prefab。
- 未保存的场景改动、运行时动态绑定和脚本逻辑引用无法识别。
- 不自动打开或切换场景；选中节点只作用于当前已打开资源。

### 当前状态
- 已完成第一版，入口为独立 `节点引用` Tab。
