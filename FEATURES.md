# 项目资源管家 Feature 规划

本文档记录 `asset-steward` 后续作为“项目资源管家”的完整功能规划。当前确认策略：

- 合并 `scene-unused-assets` 到资源管家，但先保留旧扩展不动。
- 允许删除能力，但必须预览、备份勾选和二次确认。
- 已实现资源扫描模块的低风险范围：缺失 `.meta`、孤立 `.meta`、空目录、类型统计，只预览和定位。
- 已实现资源引用检查第一版：扫描列表可一键选择资源，读取 `.meta` UUID，静态反查序列化资源引用方，只预览和定位。
- 已实现 resources 动态加载检查第一版：对照 `assets/resources` 和直接调用的 `resources.load/loadDir`，只报告和定位。
- 已实现包体贡献统计第一版：按所有子目录递归大小、扩展名大小、Top 文件和 Top 主场景递归引用文件展示源资源体积。
- 已实现目录规范检查第一版：复用自动分类规则，报告目录不符合项并支持转为移动预览。
- 已实现未引用资源扫描第一版：从主场景 UUID 依赖图生成只读候选，支持筛选和定位，不提供删除。

## 1. 资源管家框架

### 目标
- 用一个扩展承载项目资源扫描、分类、未引用资源、健康检查和报告能力。
- 保留现有自动分类和批量移动能力，不破坏已有规则和历史。

### 面板入口
- 菜单：`项目资源管家 -> 打开资源管家`
- 面板 Tab：
  - `资源扫描`
  - `自动分类`
  - `未引用资源`
  - `健康检查`
  - `历史与报告`

### 当前状态
- 已完成多 Tab 框架。
- 已迁移自动分类和历史反向计划。
- 已完成 `资源扫描` Tab 的低风险体检入口。
- 已完成资源引用检查第一版，并放在 `资源扫描` Tab 下方；扫描资源列表支持一键“查引用”。
- 已完成 resources 动态加载检查第一版，并放在 `健康检查` Tab；`loadDir` 命中的目录按整目录已使用处理。
- 已完成包体贡献统计第一版，并放在 `健康检查` Tab；默认扫描 `assets`、排除 `.meta`、显示 Top 20。
- 已新增 `get-toolbox-framework` 框架接口，统一登记各 Tab 后续模块的状态、输入、输出、安全边界和下一步。
- 未引用资源、其余健康检查模块和历史报告页已改为读取框架清单渲染模块入口，当前不执行对应删除或修复。

## 2. 资源扫描

### 目标
- 提供低风险项目资源体检入口，只展示问题，不直接修改或删除。
- 为后续分类、清理和未引用扫描提供统一扫描结果。

### 扫描范围
- 只扫描 `assets/`。
- 不扫描 `library/`、`temp/`、`build/`、`node_modules/`。
- `.meta` 不作为普通资源展示，但会参与异常判断。

### 功能
- 缺失 `.meta`：
  - 文件或目录存在，但对应 `.meta` 不存在。
  - 标记为高风险，因为 AssetDB 移动、删除和引用维护可能不可靠。
- 孤立 `.meta`：
  - `.meta` 存在，但对应资源不存在。
  - 标记为可清理候选，但第一版只展示和定位。
- 空目录：
  - 目录内没有任何文件或子目录，或只有目录自身 `.meta`。
  - 保护 `assets/res`、`assets/resources`、`assets/script`、`assets/scene`。
- 类型统计：
  - 按扩展名统计数量和总大小。
  - 单独列出目录数量、文件数量、缺失 meta 数、孤立 meta 数、空目录数。
- 路径筛选：
  - 搜索路径关键字。
  - 按扩展名筛选。
  - 按目录筛选。
- 定位：
  - 能定位到资源或目录。
  - 孤立 `.meta` 只能显示路径，定位能力依赖 Creator AssetDB 是否识别。

### 输入
- 搜索文本。
- 扩展名过滤。
- 扫描目录，默认 `assets`。

### 输出
- 扫描摘要。
- 异常列表。
- 类型统计表。
- 可导出的报告数据结构，后续交给“历史与报告”Tab。

### 安全限制
- 第一版不删除文件。
- 第一版不自动修复 `.meta`。
- 第一版不创建目录。

### 你需要做什么
- 在 Creator 中打开资源管家。
- 进入 `资源扫描` Tab。
- 点击扫描，人工复核异常项。
- 对缺失 `.meta` 的资源优先在 Creator 内重新导入或手动确认来源。

## 3. 自动分类

### 目标
- 批量把 `assets/` 下资源移动到规范目录。
- 通过 AssetDB 执行移动，保持资源和 `.meta` 一起迁移。

### 当前已实现功能
- 手动批量移动到统一目标目录。
- 按扩展名和文件名关键词自动分类。
- 冲突策略：
  - 跳过冲突项。
  - 自动重命名。
  - 覆盖目标文件。
- 移动前生成计划预览。
- 移动后可清理本次产生的空源目录。
- 保存规则和移动历史。
- 生成反向移动计划。

### 默认分类
- `.prefab -> assets/res/prefab`
- `.mp3/.wav/.ogg -> assets/res/audio`
- `.fbx/.gltf/.glb -> assets/res/model`
- UI 贴图 -> `assets/res/image/ui`
- VFX 贴图 -> `assets/res/image/vfx`
- 环境贴图 -> `assets/res/image/environment`
- 模型贴图 -> `assets/res/image/model`
- 其它贴图 -> `assets/res/image/other`
- `.effect/.chunk -> assets/res/effects`
- `.mtl -> assets/res/material`

### 安全限制
- 不允许移动 `assets` 根目录。
- 缺少 `.meta` 的源资源会阻止移动。
- 覆盖只允许文件覆盖文件。
- 覆盖必须备份勾选和二次确认。
- 目标同时是本批次源资源时阻止。
- 目录不能移动到自身子目录。
- 空目录清理只清理本次成功移动资源的原目录。

### 你需要做什么
- 正式整理前提交或备份项目。
- 优先使用“预览计划”检查分类结果。
- 对进入 `image/other` 的贴图人工复核，必要时补文件名关键词规则。
- 移动后在 Creator 中打开主场景、Prefab 和材质检查引用。

## 4. 未引用资源

### 目标
- 将现有 `scene-unused-assets` 能力合并进资源管家。
- 通过场景依赖图识别指定场景未递归引用的资源。

### 功能
- 选择目标 `.scene`。
- 选择扫描目录，默认 `assets`。
- 递归追踪场景、Prefab、材质、Effect、动画和模型 `.meta` 中的 UUID。
- 展示未引用候选资源。
- 支持定位候选资源。
- 支持删除候选资源，但必须高风险确认。

### 输入
- 目标场景。
- 扫描目录。
- 保护类型配置。

### 输出
- 扫描数量。
- 已引用数量。
- 未引用数量。
- 无法解析引用数量。
- 未引用候选列表。

### 删除策略
- 默认不选中任何删除项。
- 删除前必须勾选“已备份项目”。
- 删除前必须二次弹窗确认。
- 删除只允许删除最近一次扫描结果中的候选资源。
- 删除通过 AssetDB 执行。

### 当前状态
- 已完成第一版，入口放在 `未引用资源` Tab，原 `scene-unused-assets` 扩展保持不变。
- 默认主场景为 `assets/scene/main.scene`，默认扫描目录为 `assets/res`。
- 与 Top 主场景递归引用大文件共用同一序列化 UUID 依赖图。
- 支持候选路径、扩展名筛选和资源定位。
- `.ts/.js/.cjs/.mjs/.chunk` 强制保护，不进入候选。
- 动态加载和运行时引用无法可靠确认，所有候选均需要人工复核。
- 第一版不提供删除能力；删除策略仍为后续规划。

### 风险
- `resources.load`、AssetManager 动态路径和运行时拼接路径无法完全识别。
- Shader include、脚本模块路径和渠道动态资源需要人工复核。

### 你需要做什么
- 先选择主场景扫描。
- 人工复核 `resources` 和动态加载资源。
- 只对明确不用的候选执行删除。

## 5. 健康检查

### 目标
- 找出资源整理后的潜在风险，不直接修改资源。

### 功能候选
- 同名资源：
  - 不同目录下同名文件。
  - 帮助发现分类后命名冲突风险。
- 重复内容：
  - 对文件计算 hash。
  - 展示同 hash 文件组。
  - 第一版只报告，不删除。
- 大文件：
  - 列出超过阈值的资源。
  - 阈值默认可配置，例如 1 MB、5 MB、10 MB。
- `resources` 动态资源检查：
  - 列出 `assets/resources` 下所有资源。
  - 对照代码中的 `resources.load` 字符串路径。
  - 标记可能未被动态加载的 resources 资源。
- 材质贴图检查：
  - 扫描 `.mtl` 和材质序列化数据中的 UUID。
  - 标记缺失引用或无法解析引用。
- 目录规范检查：
  - 检查资源是否落在建议目录。
  - 例如 Prefab、Audio、Model、Image、Effect、Material。

### 输入
- 扫描目录。
- 大文件阈值。
- 是否计算 hash。

### 输出
- 健康检查报告。
- 风险等级。
- 定位按钮。

### 安全限制
- 默认只报告。
- 不自动删除重复资源。
- 不自动移动资源。

### 你需要做什么
- 对重复资源先人工确认引用关系。
- 对大文件确认是否为必要投放资源。
- 对 `resources` 下资源确认是否真的需要动态加载。

## 6. 资源引用检查

### 目标
- 在移动、删除或清理资源前，反查哪些序列化资源引用了目标资源。
- 降低误删、误移动和重复资源处理的风险。

### 功能
- 选择一个或多个资源作为目标。
- 读取目标资源及其 `.meta` 中的主 UUID 和子资源 UUID。
- 扫描 `.scene`、`.prefab`、`.mtl`、`.material`、`.anim`、`.effect` 等文本序列化资源。
- 展示引用目标 UUID 的文件列表。
- 支持按引用文件类型筛选。
- 支持定位引用方和被引用资源。
- 支持将引用方作为新目标继续“查上级”。
- `.scene/.prefab` 引用逐条展示节点路径、组件类型和字段路径。
- 对当前已打开资源中存在稳定节点 ID 的引用位置支持选中节点。

### 输入
- 被检查资源路径。
- 扫描目录，默认 `assets`。
- 引用文件类型过滤。

### 输出
- 被检查资源 UUID 列表。
- 引用方文件列表。
- 每个引用方命中的 UUID 数量。
- 场景/Prefab 引用位置、节点路径、组件类型和字段路径。
- 未找到引用时给出风险提示，而不是直接判定可删除。

### 安全限制
- 只做静态 UUID 搜索。
- 不识别代码运行时拼接路径。
- 不自动删除无引用资源。

### 当前状态
- 已完成第一版，入口放在 `资源扫描` Tab 下方。
- 已支持输入一个或多个资源路径、扫描目录和引用文件类型过滤。
- 已支持在资源扫描结果列表中直接点击“查引用”，无需手动复制路径。
- 已支持定位目标资源和引用方资源。
- 已支持引用方继续“查上级”，可重复操作追踪上上级及更高层引用。
- 已支持场景/Prefab 精确引用位置，以及“定位资源”和当前打开资源内“选中节点”。
- 暂未接入报告导出、未引用资源删除候选和动态加载路径判断。

### 你需要做什么
- 删除或覆盖资源前，先用引用检查确认引用方。
- 对未找到引用的资源，再结合未引用资源扫描和 `resources` 动态检查一起判断。

## 7. resources 动态加载检查

### 目标
- 控制 `assets/resources` 目录规模，避免可玩广告包体被动态资源入口拖大。

### 功能
- 扫描 `assets/resources` 下所有资源。
- 扫描 TypeScript/JavaScript 中直接调用的 `resources.load` 和 `resources.loadDir`。
- 提取静态字符串路径。
- 标记：
  - resources 中有资源，但代码中未发现静态加载路径。
  - 代码中加载路径可能找不到资源。
  - 使用动态拼接路径，无法静态确认。

### 输入
- 代码扫描目录，默认 `assets/script` 和 `assets/scripts`。
- resources 扫描目录，默认 `assets/resources`。

### 输出
- resources 资源列表。
- 静态加载路径列表。
- 可能未被加载的 resources 资源。
- 可能加载失败的代码路径。
- 动态拼接路径告警。

### 安全限制
- 只报告，不移动或删除 resources 资源。
- 动态路径只能提示人工复核。

### 当前状态
- 已完成第一版，入口放在 `健康检查` Tab。
- 默认只扫描 `assets/script` 和 `assets/scripts`；不存在的默认目录自动跳过。
- `resources.loadDir("目录")` 会将目录下全部资源视为静态命中。
- 支持 `SpriteFrame` 的 `/spriteFrame` 和 `Texture2D` 的 `/texture` 子资源路径匹配。
- 暂不识别封装加载器、其它 AssetManager/Bundle API 和运行时生成路径。

### 你需要做什么
- 对未被静态路径命中的 resources 资源人工确认。
- 能用编辑器引用的资源优先移出 `resources`。
- 修改代码动态路径前先确认运行时逻辑。

## 8. 包体贡献统计

### 目标
- 找出最影响包体体积的目录、类型和单文件，优先处理高收益资源。

### 功能
- 按目录统计资源大小。
- 按扩展名统计资源大小。
- 列出 Top N 大文件。
- 列出 Top N 主场景递归引用大文件，并展示一条从主场景到资源的引用链。
- 支持排除 `.meta`。
- 支持区分 `assets/resources`、`assets/res`、`assets/ArtResources` 等目录。
- 后续可接入压缩建议，例如贴图尺寸、音频格式、模型大小。

### 输入
- 扫描目录。
- Top N 数量。
- 是否包含 `.meta`，默认不包含。
- 主场景，默认 `assets/scene/main.scene`。

### 输出
- 目录大小排行。
- 类型大小排行。
- 单文件大小排行。
- 主场景递归引用大文件及引用链。
- 可导出报告。

### 安全限制
- 只统计，不修改资源。
- 大文件不代表一定应该删除，需要结合引用和玩法需求判断。

### 当前状态
- 已完成第一版，入口放在 `健康检查` Tab。
- 目录排行统计扫描根目录下所有子目录的递归总大小，扫描根目录总量单独显示在摘要中。
- 默认 Top 20，允许配置 1-200。
- 默认排除 `.meta`，可选择包含。
- 支持配置主场景，并从场景沿序列化 UUID 递归遍历 Prefab、模型、材质、贴图、动画、音频等资源。
- 仅下游被引用但上游不从主场景可达时，该孤立链不提供可达性；共享资源仍可能通过其它有效主场景链进入排行。
- `resources.load/loadDir` 即使能静态识别也不计入主场景引用排行。
- 当前统计项目源资源磁盘体积，不等同于构建后包体。

### 你需要做什么
- 优先检查 Top 大文件是否必要。
- 对大贴图、大模型、大音频分别决定压缩或替换方案。

## 9. 目录规范检查

### 目标
- 判断资源是否放在推荐目录中，并给出可复核的移动建议。

### 功能
- 复用自动分类规则。
- 对当前资源生成“建议目录”。
- 标记当前目录与建议目录不一致的资源。
- 支持一键生成移动预览计划，但仍需用户确认执行。

### 输入
- 扫描目录。
- 自动分类规则。

### 输出
- 不符合目录规范的资源列表。
- 建议目标目录。
- 可转化的移动计划。

### 安全限制
- 默认只报告。
- 生成移动计划后仍走自动分类的预览和确认流程。

### 当前状态
- 已完成第一版，入口放在 `健康检查` Tab。
- 默认扫描 `assets/res` 下所有可命中启用分类规则的文件。
- 按规则显示顺序使用首个命中项，和自动分类实际行为保持一致。
- 只列出当前目录与建议目标目录不一致的资源；未命中规则的资源只计数。
- 支持一键把不符合项转为现有自动分类移动预览。
- 目标冲突、缺少 `.meta`、重命名和覆盖仍由现有移动预览与执行流程处理。

### 你需要做什么
- 先调整分类规则，确保建议目录符合项目约定。
- 对建议移动项人工复核后再执行。

## 10. 场景和 Prefab 引用健康检查

### 目标
- 在资源整理后检查场景和 Prefab 中是否存在丢失 UUID 或无法解析引用。

### 功能
- 扫描 `.scene` 和 `.prefab`。
- 提取 UUID。
- 对照项目资源 UUID 映射。
- 标记无法解析的 UUID。
- 展示所在文件和命中次数。

### 输入
- 扫描目录，默认 `assets`。
- 文件类型，默认 `.scene` 和 `.prefab`。

### 输出
- 无法解析 UUID 列表。
- 涉及文件列表。
- 引用数量统计。

### 安全限制
- 只报告，不自动修复。
- 部分内置资源或引擎资源 UUID 可能需要加入白名单。

### 你需要做什么
- 资源大规模移动后运行一次。
- 对无法解析 UUID 的场景或 Prefab 在 Creator 中打开确认。

## 11. 历史与报告

## 14. 资源引用扩展

### 目标
- 在现有“目标资源 -> 直接引用方”结果上继续向上追踪，帮助确认资源最终由哪些场景、Prefab 或其它资源使用。
- 当引用发生在场景或 Prefab 内时，将结果精确到节点、组件和字段，而不是只展示引用方文件。

### 功能
- 引用方操作栏除“定位”外，新增“查上级引用”：
  - 对当前引用方再次执行引用检查，展示它的直接上级引用方。
  - 可继续查询上上级及更高层引用，行为与主场景递归引用链一致。
  - 每次查询替换当前目标，用户可按 `贴图 -> 材质 -> Prefab -> 场景` 顺序逐级追踪。
- 场景和 Prefab 引用详情：
  - 引用方为 `.scene` 或 `.prefab` 时，解析序列化数据中的引用位置。
  - 展示引用所在的节点名称和节点层级路径。
  - 展示引用所在的组件类型。
  - 展示组件中引用目标资源的字段路径。
  - 同一文件内存在多处引用时逐条展示，不只汇总命中次数。
- 引用详情示例：
  - `assets/scene/main.scene`
  - 节点：`Canvas/HUD/FishIcon`
  - 组件：`cc.Sprite`
  - 字段：`_spriteFrame`

### 输出
- 直接引用方列表。
- 每个引用方的上级引用入口，可重复操作逐级追踪引用链。
- 场景或 Prefab 内的节点路径、组件类型、字段路径和命中 UUID。
- 无法解析到具体节点或组件时，保留文件级结果并标记“仅文件级可定位”。

### 安全限制
- 只解析文本序列化资源中的静态 UUID 引用，不修改场景、Prefab 或资源。
- 代码动态加载、运行时拼接路径和引擎内部引用无法精确到节点、组件或字段，需要人工复核。
- “选中节点”只查询当前打开资源，不自动打开或切换场景/Prefab。
- Prefab 实例覆盖和部分序列化对象没有稳定节点 ID 时，只提供引用位置和资源定位。

### 当前状态
- 已按 `1A、2C` 完成第一版。
- 引用方支持逐级“查上级”；场景/Prefab 引用逐条展示节点、组件和字段。
- 已提供“定位资源”和当前打开资源内“选中节点”，不会自动切换场景。

## 15. 场景节点引用检查

### 目标
- 从“场景里的某个节点”反查有哪些组件属性引用了它。
- 帮助删除、重命名、移动或拆分节点前确认依赖位置。

### 功能
- 输入节点 ID；输入为空时使用当前 Creator 选中的节点。
- 扫描目录默认 `assets`，文件类型默认 `.scene,.prefab`。
- 在扫描到的场景或 Prefab 中按节点 `_id` 定位目标节点。
- 只统计组件对象中的字段引用，过滤组件自身 `_node` 挂载关系。
- 展示引用组件的节点路径、组件类型、字段路径和目标节点路径。
- 支持定位资源；对应资源已打开时，可选中引用组件所在节点或目标节点。

### 安全限制
- 只读解析磁盘上的 `.scene/.prefab` 序列化 JSON，不修改场景或 Prefab。
- 未保存的场景改动、运行时动态绑定和脚本逻辑引用无法识别。
- 不自动打开或切换场景；选中节点只作用于当前已打开资源。

### 当前状态
- 已完成第一版，入口放在 `健康检查` Tab。
