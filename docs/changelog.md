# 变更记录

本文档记录面向发布和交付的用户可见变化。更细的规划背景仍保留在 [`../FEATURES.md`](../FEATURES.md)，优化进度见 [`optimization-roadmap.md`](optimization-roadmap.md)。

## 1.13.0

### 新增

- 新增 `总览` 页，聚合已运行模块风险、推荐下一步和危险操作状态。
- 新增 `工具面板`，可按项目隐藏或显示健康检查功能块。
- 新增资源扫描、资源引用检查、场景节点引用检查和场景/Prefab 引用健康检查。
- 新增 resources 动态加载检查、包体贡献统计、目录规范检查、材质贴图检查和重复资源检查。
- 新增未引用资源扫描、删除预览、备份 manifest 和删除执行审计。
- 新增当前会话 Markdown/JSON 报告导出。
- 新增项目缓存清理入口，仅允许清理项目内 `library` 和 `temp`。

### 改进

- 原 `健康检查` 聚合页拆为 7 个独立功能页：节点引用、resources 加载、包体统计、目录规范、材质贴图、重复资源和场景引用。
- 工具面板开关现在会同时隐藏对应功能页和顶部 Tab；关闭当前功能时自动返回总览。
- 主进程返回兼容协议元信息：`ok`、`protocolVersion`、`warnings`。
- 错误返回统一为 `{ ok: false, error }`，并开始区分 validation、not-found、conflict、permission、external 等错误大类。
- 移动历史会记录失败移动明细、空目录清理结果和覆盖风险。
- 面板布局支持多个 Resizable 分栏和包体统计布局预设。
- 面板代码拆分为请求、格式化、布局、总览、健康检查、历史、未引用、扫描、节点引用、自动分类、工具面板和对应 render 模块。
- 文档拆分为用户手册、开发手册、架构说明、安全边界、优化路线图和变更记录。

### 安全

- 移动、覆盖、删除和缓存清理均保留预览或确认步骤。
- 未引用删除必须先创建备份，并写入 `manifest.json` 与 `execution-result.json`。
- 会话报告不会写入移动计划临时执行 token。
- 健康检查默认只报告和定位，不自动修改资源。

### 验证

- 当前自动化测试命令：`npm test`。
- 当前测试覆盖协议、移动计划、引用图、未引用删除、项目维护、报告导出、健康检查、面板模型和 DOM 渲染 helper。
