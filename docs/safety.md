# 安全边界

`asset-steward` 的默认策略是先预览、再人工复核、最后通过 Cocos Creator AssetDB 执行。

## 只读能力

以下能力只报告和定位，不修改资源：

- 资源扫描。
- 资源引用检查。
- 场景节点引用检查。
- resources 动态加载检查。
- 包体贡献统计。
- 目录规范检查的检查阶段。
- 材质贴图检查。
- 重复资源检查。
- 场景和 Prefab 引用健康检查。

## 移动能力

资源移动必须先生成预览计划。执行前会再次检查：

- 源资源存在。
- 源资源存在 `.meta`。
- 目标目录存在或由规则模式创建。
- 冲突策略仍然有效。
- 覆盖操作已人工确认项目备份。

移动和覆盖通过 Creator AssetDB 执行。覆盖会先删除目标文件，反向计划不能恢复被覆盖前的目标文件。

## 未引用删除能力

未引用候选不代表一定可删除。执行删除前需要：

- 候选由当前主场景 UUID 依赖图重新校验。
- 用户人工勾选候选。
- 用户确认已复核动态加载风险。
- 工具成功创建备份。
- 二次弹窗确认。

备份保存到 `backups/asset-steward/unused-delete-时间戳/`。`manifest.json` 会记录备份文件、源路径、大小和 SHA-256。执行后会写入 `execution-result.json`，记录删除成功和失败明细。

## 项目缓存清理

项目维护里的缓存清理只允许删除项目根目录下固定白名单目录：

- `library`
- `temp`

执行前需要面板弹窗确认，主进程也会要求 `confirmed: true`。清理逻辑会解析真实路径并拒绝项目外路径、嵌套路径和非白名单目录；不会删除 `assets`、`profiles`、`settings`、`local` 或其它项目文件。

清理后的项目处理方式由面板枚举选择：

- `2A`：只尝试刷新 AssetDB，并提示必要时手动重新打开项目。
- `2B`：直接尝试编辑器级项目重载/重启能力；如果当前 Creator API 不支持，会返回错误提示。

## 历史与审计

新执行的移动历史会持久化：

- 成功移动项。
- 失败移动明细。
- 删除空源目录结果。
- 清理失败目录明细。
- 是否包含覆盖风险。

旧历史记录可能没有失败明细，面板会继续提示查看当次运行日志。
