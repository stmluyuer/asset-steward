# 用户手册

本文档面向日常使用 `asset-steward` 的项目成员，说明每个功能页能做什么、使用顺序和安全边界。

## 打开方式

1. 在 Cocos Creator 扩展管理器启用 `asset-steward`。
2. 点击顶部菜单 `项目资源管家 -> 打开资源管家`。
3. 默认进入 `总览` 页；原健康检查能力已拆成 `节点引用`、`resources 加载`、`包体统计`、`目录规范`、`材质贴图`、`重复资源`、`场景引用` 7 个独立页。

## 推荐流程

1. 先看 `总览`，确认当前已有结果、风险分组和推荐下一步。
2. 跑 `资源扫描`，优先处理缺失 `.meta`、孤立 `.meta` 和空目录。
3. 跑 `资源引用检查`、`节点引用` 或 `场景引用`，确认移动/删除前的静态引用风险。
4. 需要整理目录时进入 `自动分类`，先生成移动预览，再人工复核计划。
5. 需要删除未引用候选时进入 `未引用资源`，先扫描、筛选、预览删除，再确认备份和二次弹窗。
6. 真实整理后进入 `历史与报告` 导出 Markdown/JSON 报告。

## 总览

总览会聚合已经运行过的模块结果，包括资源结构异常、未引用候选、resources 动态加载风险、重复资源、材质贴图、场景/Prefab 引用健康、当前移动计划、删除预览、最近执行和运行日志告警。

- `风险`：当前最需要复核的结果。
- `推荐下一步`：根据缺失结果和最高风险生成入口。
- `危险操作`：移动计划、删除预览、最近执行和日志状态。
- `一键健康检查`：按当前工具开关运行若干只读检查。

## 资源扫描

资源扫描只读，不移动、不删除、不修复 `.meta`。

它会扫描 `assets/` 或指定子目录，展示：

- 缺失 `.meta` 的资源或目录。
- 孤立 `.meta`。
- 空目录。
- 资源列表和基础状态。

建议先处理缺失 `.meta`，因为 AssetDB 移动、删除和引用维护都依赖 `.meta`。

## 资源引用检查

资源引用检查读取目标资源 `.meta` 中的 UUID，并静态搜索 `.scene`、`.prefab`、材质、动画、Effect 等序列化文件。

注意：

- 未找到引用不代表资源一定可删。
- 动态加载、运行时拼接路径和脚本逻辑不会被完全识别。
- `.scene/.prefab` 引用可展示节点路径、组件和字段；只有当前打开资源中有稳定节点 ID 时才可选中节点。

## 自动分类和移动

自动分类支持两种模式：

- `手动移动`：勾选资源后移动到一个目标目录。
- `规则分类`：按扩展名和文件名关键词匹配目标目录。

执行前必须先生成预览计划。计划会展示源路径、目标路径、操作类型、阻止原因、重命名和覆盖情况。

安全边界：

- 不允许移动 `assets` 根目录。
- 缺少 `.meta` 的资源会被阻止。
- 覆盖前必须确认已经备份项目。
- 覆盖会先删除目标文件，反向计划不能恢复被覆盖前的目标。
- 执行期间源资源或目标状态变化会导致失败，需要重新扫描和预览。

## 未引用资源

未引用资源扫描从主场景出发，沿序列化 UUID 依赖图递归追踪，列出没有到达的候选资源。

候选不等于可删除。删除前需要：

- 人工筛选候选。
- 复核 resources 动态加载、AssetManager、运行时拼接路径、Shader Chunk 和渠道资源。
- 生成删除预览。
- 确认备份范围。
- 二次弹窗确认。

删除执行会先备份到 `backups/asset-steward/unused-delete-时间戳/`，写入 `manifest.json` 和执行后的 `execution-result.json`。

## 独立检查功能

原健康检查页已经拆为 7 个独立功能页：

- `节点引用`：反查场景或 Prefab 中引用目标节点的组件属性。
- `resources 加载`：静态检查 `resources.load/loadDir` 调用和疑似未加载资源。
- `包体统计`：统计目录、类型、大文件和主场景递归可达资源体积。
- `目录规范`：按自动分类规则报告目录不一致资源，并可生成移动预览。
- `材质贴图`：检查材质贴图 UUID、解析状态和主场景可达性。
- `重复资源`：检查跨目录同名资源和 SHA-256 相同内容。
- `场景引用`：检查场景和 Prefab 中无法解析的序列化 UUID。

这些检查默认只报告和定位，不直接修改资源。顶部 `工具面板` 可按项目隐藏或显示对应功能页及其 Tab。

## 历史与报告

历史与报告页支持：

- 查看最近 30 条移动历史。
- 查看单次移动成功项、失败项、空目录清理结果和覆盖可恢复状态。
- 生成反向移动计划。
- 查看运行日志。
- 导出当前会话 Markdown/JSON 报告到 `reports/asset-steward/`。

报告不会重新运行扫描，也不会写入当前移动计划的临时执行 token。

## 安全建议

- 执行移动、覆盖、删除或缓存清理前，先提交版本库或备份项目。
- 对缺失 `.meta`、无法解析 UUID、动态加载资源和重复资源优先人工复核。
- 移动后打开关键场景、Prefab、材质做引用回归。
- 删除前不要只看“未引用候选”，还要检查代码和运行时加载路径。

更详细的安全边界见 [`safety.md`](safety.md)。
